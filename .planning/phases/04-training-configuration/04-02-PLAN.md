```python
# At top of file, add import:
from lightning.pytorch.loggers import WandbLogger

# In on_validation_epoch_end, AFTER self._plot_and_save(cm_tensor, epoch):
# Log confusion matrix image to WandB if WandbLogger is active
save_dir = self.output_dir / f"epoch_{epoch:03d}"
png_path = save_dir / "confusion_matrix.png"
for lgr in trainer.loggers:
    if isinstance(lgr, WandbLogger):
        lgr.log_image(
            key="confusion_matrix",
            images=[str(png_path)],
            step=trainer.current_epoch,
        )
        logger.debug(f"Logged confusion matrix to WandB (epoch {epoch})")
```

Key points:
- `WandbLogger.log_image()` accepts file path strings — no need to import `wandb` directly.
- Use `isinstance` guard so callback works with or without WandB (graceful degradation).
- The PNG file is already saved by `_plot_and_save` before this code runs.
- Use `trainer.loggers` (list), not `trainer.logger` (single), to handle multiple loggers.

NOTE: The `_plot_and_save` method currently does not return the `png_path`. Look at the method to confirm it saves to `self.output_dir / f"epoch_{epoch:03d}" / "confusion_matrix.png"` — if so, reconstruct the path. If the path pattern is different, match it exactly.
